Why is there a 651 error (Rostelecom) when connecting to the Internet?

Subscribers of the Rostelecom provider, which provides Internet access services using a connection via ADSL modems, often enough when they try to connect (dial up) receive a message that a Rostelecom 651 modem error has occurred. Many immediately start to panic and call Customer Support. Do not do this, because if the appearance of the failure is not connected specifically with the provider, in most cases you can fix it yourself. The following is a short instruction on what actions can be taken in different situations.

Why does Rostelecom error 651 occur?

We‚Äôll talk about methods of troubleshooting a bit later, but for now, let's consider what could have caused it to appear, because, it would seem, yesterday everything worked like a clock.





Most of the experts among all possible situations distinguish the following:

  • blocking the connection with the protective components of the system and antiviruses;
  • incorrect modem settings;
  • ‚ÄúGathering‚Äù or obsolescence of network card drivers;
  • improper modem connection and user data entered at login;
  • Conflicts between multiple network cards (if installed)
  • malfunctions in the connection used;
  • viral exposure;
  • problems on the part of the provider.

Based on the types of these situations, you can make a specific decision about troubleshooting.

Error "Rostelecom" 651: what to do first?

In general, it is believed that when such failures occur, all network equipment is subject to reboot. This is an axiom. And the connection error 651 by Rostelecom in this case is no exception to the rule.

It is possible that a short-term malfunction occurred at the software or hardware level. Just reboot the modem and the computer device. It is advisable not to reset the modem, but disconnect it from the power supply for some time.

Anti-virus blocking

Quite often it is noted that error 651 by Rostelecom when connecting to the Internet may also be due to the fact that the installed full-time antivirus program or even its own Windows firewall simply blocks the connection.





As already clear, the way out of this situation for a start is to turn off the antivirus, say, for about ten minutes and then check the possibility of establishing a connection. If the Rostelecom error 651 appears again, completely deactivate the firewall in the system settings, although this is not recommended. But do not be afraid. Nothing wrong with that. If it turns out that the blocking is carried out precisely by these two components, leave the firewall disabled and change the anti-virus software.

Setting the correct modem settings

But these were only special cases. This is usually not the problem. For the most part, the appearance of the Internet error "Rostelecom" 651 is associated with incorrect options installed in the settings of the router.





In this case, you must enter the modem‚Äôs web interface through any Internet browser and check the settings are correct. To do this, use the WAN or Internet section and check the operation mode. It must be installed on Bridge.

As an additional measure, contact support and specify the VPI and VCI parameters. If necessary, make the appropriate adjustments and save the settings.

If the router is in router mode, check that the connection type is set to PPP over Ethernet (PPPoE).





Driver issues

In case of the repeated appearance of the Rostelecom 651 error, it is necessary to check the status of the drivers of the network adapter. It is possible that they, as they say, "flew" or corny outdated.

First, call the ‚ÄúDevice Manager‚Äù (you can use the Run menu, which contains the line devmgmt.msc), then find your network card and, through the PCM menu or the properties on the driver tab, update the control software. If this does not help, completely remove the device from the system, and reinstall the drivers using not the Windows database, but the original disk that was delivered when you purchased the computer device. If there is no such drive, use directory programs like DriverPack Solution or SlimDrivers.





FTTB and GPOE connection

If the Rostelecom 651 error appears during this type of connection when connecting to the optical line through a twisted pair cable, you need to check that the cable is connected to the port of the WAN router, and not to the LAN. In addition, make sure that the entered username and password are correct.

Problems installing multiple network cards

Another common reason for the appearance of the Rostelecom 651 error is the presence on the computer device of several network cards, which can cause conflicts between equipment and its control part. The operating system simply cannot choose which device to use as the main one, referring to the drivers of one or the other.

To fix the problem, re-enter the ‚ÄúDevice Manager‚Äù and disconnect one of the network cards using the corresponding item in the RMB menu. After that, make sure that the settings for the primary BIOS and UEFI are set to the same parameters.

Connection Failures

If the above methods have not given the desired effect, use the ‚ÄúControl Panel‚Äù, go to the network connection section and delete the one that is currently in use.





After that, in the same section, create a similar new connection, specifying the parameters that were used before the failure.

Exposure to viruses

Finally, the most unpleasant thing is the penetration of various viruses to the computer, which are also able to change network settings, affect the operating system or disrupt the operation of the same drivers.

To scan your computer, use portable utilities (not the stationary antivirus installed on the system). For these purposes, programs like KVRT or Dr. Web CureIt. The best option is to use applications called Rescue Disk. They differ favorably in that they have their own bootloaders, and when you start from removable media, you can boot from them and scan before the main operating system is launched.

Provider Issues

Finally, if the problem is specifically with the provider, for example, with technical work, contact the support team and specify when the equipment will be restored to working condition.

Brief summary

In principle, the above methods in most cases eliminate the occurrence of the described error almost always allow. The issues related to equipment breakdown were not considered here, since in such a situation you will have to contact some service center and check the modem‚Äôs performance. In addition, it is sometimes advisable to check incoming connections. Perhaps somewhere the cable that goes into the router is interrupted. Sometimes there are situations when the modem ‚Äúloses‚Äù the telephone line signal, but this is rare.

In any case, first of all, pay attention to rebooting all the equipment, setting the correct options and settings for the connection used and, if necessary, update or reinstall the drivers of the network adapter (you can automatically update it with friends who have an Internet connection, using Driver Booster Utility).
