1C server installation and setup at the enterprise

In the article, we will consider instructions for installing the 1C: Enterprise server in the office. The 1C installation option in the client-server form is the most suitable. But first you need to understand what a client-server architecture is. After that, we will tell you in what cases it makes sense to implement this architecture, and most importantly - how to do it right. If you are faced with the installation for the first time, then our material will be very useful to you.

What is a client-server architecture

It is worth noting that 1C in all cases works as a client-server. But there is a so-called file version, in which all work takes place exclusively on local computers. At the same time, the system itself divides the entire computer memory into a conditionally server and conditionally client. In this case, its own built-in database management system is used. It has one drawback - low speed and low stability.

When working in the client part, requests to 1C are generated, after which they are transferred to the conditionally server part, where processing takes place. The results of this processing go back to the conditional client part. In fact, this is a server version of the system, but it is customary to call it file. As you can see, there is subtlety, but it is small. In general, it is much better to use the server version of the software, since it is much more stable and allows more clients to work with one database at the same time.

As for the client-server option, then for the proper functioning of the database, third-party software systems are used. These are MS SQL, Oracle, DB, DB2, PostgresSQL. It is worth noting that third-party control systems are considered more powerful than integrated ones. At the same time, the best performance is ensured, and most importantly, the database security is increased significantly.

We will consider installation of client-server architecture on MS SQL. This is the best option for the correct operation of 1C. When installing PostgreSQL 1C server, there is a need for databases to be indexed frequently. Oracle has a large number of problems with organizing data in a database. DB2, when compared with competitors, is limited in number length.

Do I need to transfer the database to the server version

In the event that no more than 7 people work with one database, and its size is quite small, then it will be quite enough to use the file option. But if from 8 to 12 people work simultaneously with one base, and its volume is up to 4 GB, then it is best to use the client-server version. In this case, a lot will depend on personal perception when working with the database. Problems can occur with a large number of simultaneous resource requests.

The question is that during operation, each user accesses the registers, and access to them may be blocked for other employees. Therefore, they will be queued when trying to access a register that has been locked. 1C at the same time freezes.

In the event that with a large number of calls the system freezes, it is necessary to modernize the architecture, use a more advanced type. If it works with the database for more than 15 people, and its total size is more than 4 GB, then you should not even think about installing the file system. Immediately you need to install the client-server version 1C.

  2. Installation on all computers of clients 1C. You need to install only where you need to connect to the database server.
