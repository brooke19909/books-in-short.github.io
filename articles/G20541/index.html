How to set up a local network: step-by-step instructions

If earlier local networks were considered such a means of combining computers among themselves, which was used exclusively in small and large offices or at enterprises, then with the current development of network technologies, you can create your own network in two accounts at home. Next, we will consider how to configure a local network in two versions, which can be called the simplest and do not require special knowledge when creating and configuring. We will take Windows systems as a basis, since in our country they are the most common.

How to configure a local computer-to-computer network: selecting or changing a workgroup

So where to start? First of all, you need to understand that all computers that will connect to the future network must be in the same workgroup, otherwise they will not be recognized when connected to each other. By default, Windows mainly uses two types of names: MSHOME or WORKGROUP, although the names of the computers themselves can be changed at will.

To configure a local area network between computers, you must first set the same workgroup name on all stationary terminals and laptops. In principle, you can access the settings through the "Control Panel", where the system and security section is selected, and then go to the "System" section. However, as practice shows, the easiest way is to use the Run console, which introduces the abbreviation sysdm.cpl (for versions of Windows from the seventh to the tenth).

In the window that appears, we are interested in the computer name field. Enter any name, then click the edit button for the existing default group and enter a name for the new group that is different from the default one. In both cases, use only the Latin alphabet. Such actions must be performed on all terminals (so that the group name is the same), after which all of them must be reloaded.

IPv4 Protocol Settings

In order to configure the local network in the 7th version of the system and above, you need to check the IPv4 protocol settings. To do this, go to the network and sharing section in the ‚ÄúControl Panel‚Äù, select the change in the properties of the network adapter, and use the settings of the specified protocol in the properties themselves.

Usually, all the parameters present there are set to receive automatically. But in our case, it is better to set static addresses, which on all connected computers should differ from each other by the last value. So, for example, if an internal address is set for one terminal, starting with a combination of 192.168, followed by 0 and 5, for the other, you can use the seven as the last value. And so for all computers (values ‚Äã‚Äãcan vary between 1-255). Obtaining addresses of DNS servers can be left in automatic mode, but it is necessary to disable the use of proxies for local addresses in additional parameters, if this option is activated.

Setting the visibility of computers on the network

Now the solution to the question of how to configure the local network implies taking actions that will allow computers to become visible on the network so that you can exchange files and folders, watch movies or play on the network.

To do this, additional access options are selected in the same network management section, and for all terminals, the network discovery resolution item is activated with the activation of the automatic configuration line on network devices.

After that, go to additional parameters, select "All networks" and deactivate password-protected sharing.

Sharing directories

As you can see, setting up a local network on Windows 7 or higher is quite simple. However, this is not all. Now, after all, you need to make public terminals visible on the terminals, where files that can be exchanged are stored.

Select the folder you want, through RMB use the properties, go to the access tab, click the advanced settings button. Now check the box next to access permission and check the necessary privileges. Save the changes, go to the security tab, click the change button, and in the new window - add.

Indicate the name of the computer or group (usually ‚ÄúAll‚Äù) and check the boxes for the permissions that were used in the previous version. Save the changes again, and after that, in the "Explorer" the selected directory and its contents will become visible. On this setting can be considered complete.

How to configure a local network through a router: check connected devices

Now a few words about setting up a local network using a router. The first step is to check if there is a connection between devices connected to the router based on a wireless connection.

How to configure a local network? In this case, we need a router only to determine the addresses of connected devices, and its basic parameters can be left without touching. First, enter the router‚Äôs web interface through any browser and enter 192.168.0.1 or 1.1 in the address bar, go to the DHCP and DHCP Clients List sections. The list will show all connected computers with their IP addresses. Remember the address of the terminal you want to verify.

Now call the command line (cmd), enter the ping command, and after it, the space for the desired address. If packet exchange has begun, you can continue.

Create a home group

As in the previous network configuration option, we check the workgroup, and if necessary, change it. Now in the network management section, select the network that has the status of home. If you have a public network installed, change its type.

After that, on the right, click the readiness for creation hyperlink, and in the new ‚ÄúWizards‚Äù window, use the create home group button.

Select objects to share

Now the solution to the question of how to configure the local network implies the choice of objects to which it is supposed to open public access. In the settings window that appears, they will be categorized (images, music, videos, printers, etc.).

Tick ‚Äã‚Äãoff what is required, and after going to the next window, click the "Finish" button. Next, you will need to configure shared access to the selected folders, but we will not consider this issue, since the procedure is completely similar to that described above.

Additional tips

Finally, a few words about additional measures, the use of which in some cases allows you to fix a lot of problems and eliminate problems. If the IPv4 protocol settings are set correctly, but there is no connection between the terminals, and the DNS addresses are set to automatic, try using free Google combinations for them, consisting of eights and fours (for example, four eights for the main server, for alternative - two eights and two fours).

In some cases, if the provider does not support IPv6 protocol maintenance due to the lack of a sixth version DHCP server, its use should be disabled by unchecking the corresponding line in the network adapter settings.

In the case of a wireless connection based on a router, a problem may arise: ping verification through the command console does not work. As a rule, antiviruses are at fault (at least ESET software products can block this). Thus, before specifying a ping, just temporarily disable the installed antivirus.

Conclusion

Here, in fact, is all about how to set up a local network. It seems that neither in the first nor in the second case any difficulties should arise. But which option to prefer? Apparently, creating a network based on a wireless connection is simpler. However, its only problem is that the router supports a limited number of simultaneous connections, and when creating a network, as described in the first version, there are no such conventions.
